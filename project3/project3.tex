\noindent \textbf{Task 2 - Operational Semantics}

\textbf{Example of a program that is not safe}:
\begin{center}
    \begin{verbatim}
      bool b;
      int y;
      int z;
      
      fscanf(stdin_hi, "%b", &b);
      fscanf(stdin_hi, "%d", &y);
      
      if (b) {
          fscanf(stdin_lo, "%d", &y);   
      }
      
      fscanf(stdin_lo, "%d", &z);
      fprintf(stdout_lo, "%d", z);
    \end{verbatim}
\end{center}

\noindent By Definition 2 in handout, a program is safe when public inputs are the same, public outputs are also same even 
secrete inputs can be different. 

To show our program above is not saft, then, we need to run the program twice, with two different high security input. We can 
do this in our program by changing boolean value b from true to false, then compare the output result. 

Looking at the derivation on the next page, we have the following result using same low input in $chs_0$ and $chs_1$

Initially, both executions start from equivalent channels with $in^L = [1,2]$ and empty $out^L$.  

When $b = true$, we enter the condition, and read y from $in^L$ before we read $z$.  
So the first element $1$ is assigned, leaving $in^L = [2]$, we then assigns $z = 2$, so $out^L = [2]$.  

However, if $b = false$, we skip reading y. The read of $z$ come from the first element directly, which is $z = 1$. leaving $in^L = [2]$, so $out^L = [1]$.  

Therefore, while initial chs are same, the low outputs are different once we change high input

\[
  \begin{array}{rcl}
  chs_0 &=& \langle [1,2], [\,], [true,1], [\,] \rangle \\[2mm]
  chs_1 &=& \langle [1,2], [\,], [false,1], [\,] \rangle \\[2mm]
  chs_0' &=& \langle [\,], [2^L], [\,], [\,] \rangle \\[2mm]
  chs_1' &=& \langle [2], [1^L], [\,], [\,] \rangle
  \end{array}
  \]
  \[
  chs_0 \sim chs_1 \text{but }
  chs_0' \not\sim chs_1'
  \]

  \begin{figure}
    \[
    \begin{nd}
    \der{
      ss \vdash_p \_,\, chs_0 \Rightarrow \eta_0,\, chs_0' 
      \qquad
      ss \vdash_p \_,\, chs_1 \Rightarrow \eta_1,\, chs_1'
    }{
      % First run
      \subder{
        ss \vdash_p \_,\, chs_0 \Rightarrow \eta_0,\, chs_0'
      }{
        \axiom{
          \texttt{fscanf}(stdin_{hi},\dots,\&b)
          \vdash_p \_,\, chs_0
          \Rightarrow \{b \mapsto true^H\},\, chs_0^1
        }
        \axiom{
          \texttt{fscanf}(stdin_{hi},\dots,\&y)
          \vdash_p \{b \mapsto true^H\},\, chs_0^1
          \Rightarrow \{b \mapsto true^H,\; y \mapsto 1^H\},\, chs_0^2
        }
        \subder{
          \texttt{if}(b)\{\texttt{fscanf}(stdin_{lo},\dots,\&y)\}
          \vdash_p \{b \mapsto true^H,\; y \mapsto 1^H\},\, chs_0^2
          \Rightarrow \{b \mapsto true^H,\; y \mapsto 1^H\},\, chs_0^3
        }{
          \axiom{
            \{b \mapsto true^H,\; y \mapsto 1^H\} \vdash_p b \Downarrow true^H
          }
          \axiom{
            \texttt{fscanf}(stdin_{lo},\dots,\&y)
            \vdash_{p,H} \{b \mapsto true^H,\; y \mapsto 1^H\},\, chs_0^2
            \Rightarrow \{b \mapsto true^H,\; y \mapsto 1^H\},\, chs_0^3
          }
        }
        \axiom{
          \texttt{fscanf}(stdin_{lo},\dots,\&z)
          \vdash_p \{b \mapsto true^H,\; y \mapsto 1^H\},\, chs_0^3
          \Rightarrow \{b \mapsto true^H,\; y \mapsto 1^H,\; z \mapsto 2^L\},\, chs_0^4
        }
        \axiom{
          \texttt{fprintf}(stdout_{lo},\dots,z)
          \vdash_p \{b \mapsto true^H,\; y \mapsto 1^H,\; z \mapsto 2^L\},\, chs_0^4
          \Rightarrow None,\, chs_0'
        }
      }
    
      % Second run
      \subder{
        ss \vdash_p \_,\, chs_1 \Rightarrow \eta_1,\, chs_1'
      }{
        \axiom{
          \texttt{fscanf}(stdin_{hi},\dots,\&b)
          \vdash_p \_,\, chs_1

        }
        \axiom{
          \texttt{fscanf}(stdin_{hi},\dots,\&y)
          \vdash_p \{b \mapsto false^H\},\, chs_1^1
          \Rightarrow \{b \mapsto false^H,\; y \mapsto 1^H\},\, chs_1^2
        }
        \subder{
          \texttt{if}(b)\{\texttt{fscanf}(stdin_{lo},\dots,\&y)\}
          \vdash_p \{b \mapsto false^H,\; y \mapsto 1^H\},\, chs_1^2
          \Rightarrow \{b \mapsto false^H,\; y \mapsto 1^H\},\, chs_1^2

\textbf{Task 3}

\textbf{Update Domain}

To make security errors explicit in the values in the semantics, we need to add SecErr value
The old domain for value is Val = PrimVal × SecLab , and the new domain for value will be Val = PrimVal × SecLab + {SecErr}

Similarly, the new domain for frame will be Frame = EnvBlock × Val + {SecErrFrame}
\\

\textbf{Update Operational Semantics}

\[
\frac{
  \rho s, chs_0 \vdash_{p,\lambda^*} e_0 \Downarrow SecErr, chs_1
}{
  \rho s, chs_0 \vdash_{p,\lambda^*} e_0 + e_1 \Downarrow SecErr, chs_1
}
\;\textsc{Op(+) left error}
\]

\[
\frac{
  \rho s, chs_0 \vdash_{p,\lambda^*} e_0 \Downarrow v, chs_1
  \quad
  \rho s, chs_1\vdash_{p,\lambda^*} e_1 \Downarrow SecErr, chs_2
}{
  \sigma \vdash_{p,\lambda^*} e_0 + e_1 \Downarrow SecErr, chs_2
}
\;\textsc{Op(+) right error}
\]

\[
\frac{
  \rho s, chs_i \vdash_{p,\lambda^*} e_i \Downarrow SecErr, chs_{i+1}
}{
  \rho s, chs_0 \vdash_{p,\lambda^*} f(e_0,\dots,e_i,\dots,e_{n-1}) \Downarrow SecErr, chs_{i+1}
}
\;\textsc{Call SecErr Arg}
\]

\[
\frac{
  ss \vdash_{p,\lambda^*} [\vec{x}\mapsto \vec{v^\lambda}], chs_0 \Rightarrow SecErrFrame, chs_1
}{
  \rho s, chs_0 \vdash_{p,\lambda^*} f(\vec{e}) \Downarrow SecErr, chs_1
}
\;\textsc{Call SecErr Body}
\]

\[
\frac{
  \rho s, chs \vdash_{p,\lambda^*} e \Downarrow v^{\lambda_0}, chs'
  \quad
  \lambda_0 \not\sqsubseteq \lambda
}{
  \rho s, chs \vdash_{p,\lambda^*} \texttt{fprintf}(stdout_{\lambda},\dots,e) \Downarrow SecErr, chs'
}
\;\textsc{Fprintf SecErr}
\]

\[
\frac{
  \rho s, chs \vdash_{p,\lambda^*} e \Downarrow SecErr, chs'
}{
  \rho s, chs \vdash_{p,\lambda^*} \texttt{fprintf}(stdout_{\lambda},\dots,e) \Downarrow SecErr, chs'
}
\;\textsc{Fprintf Propagation}
\]

\[
\frac{
  \lambda^* \not\sqsubseteq \rho s(x)
}{
  x = e \vdash_{p,\lambda^*} \rho s, chs \Rightarrow SecErrFrame, chs
}
\;\textsc{AssignNSU}
\]


\[
\frac{
  \rho s, chs \vdash_{p,\lambda^*} e \Downarrow SecErr, chs'
}{
  x = e \vdash_{p,\lambda^*} \rho s, chs \Rightarrow SecErrFrame, chs'
}
\;\textsc{Assign Propagation}
\]

\[
\frac{
  x \in \mathrm{dom}\,\rho s
  \quad
  \lambda^* \not\sqsubseteq \rho s(x)
}{
  \texttt{fscanf}(stdin_{\lambda},\dots,\&x)\vdash_{p,\lambda^*}
  \rho s, chs
  \Rightarrow
  SecErrFrame, chs
}
\;\textsc{FscanfNSU}
\]

\[
  \frac{
    x \in \mathrm{dom}\,\rho s
    \quad
    \lambda^* \sqsubseteq \rho s(x)
    \quad
    \lambda^* \not\sqsubseteq \lambda
  }{
    \texttt{fscanf}(stdin_{\lambda},\dots,\&x)\vdash_{p,\lambda^*}
    \rho s, chs
    \Rightarrow
    SecErrFrame, chs
  }
  \;\textsc{Fscanf Propagation}
\]

\[
\frac{
  \rho s, chs \vdash_{p,\lambda^*} e \Downarrow SecErr, chs'
}{
  \texttt{if }(e)\; s_0\; \texttt{else }\; s_1
  \vdash_{p,\lambda^*}
  \rho s, chs
  \Rightarrow
  SecErrFrame, chs'
}
\;\textsc{IfTrue SecErr}
\]

\frac{

\textbf{Update Equivalence Definition}


we extend the definition 1 of equivalence on the handout with following cases
\begin{enumerate}
  \setcounter{enumi}{0}
  
  \item
  two value are equivalent if
  \[
  SecErr \sim SecErr
  \]
  
  \item
  two frame are equivalent if
  \[
  SecErrFrame \sim SecErrFrame
  \]

  \item 
  all original definition of equivalence
  \end{enumerate}


\textbf{Example of a program that is not safe}:
\begin{center}
    \begin{verbatim}
      bool b;
      fscanf(stdin_hi, "%b", &b);
      if (b) {
          fprintf(stdout_lo, "secret");
      }
    \end{verbatim}
\end{center}

We execute this program twice with b = true and b = false, then compare the output. Let $chs_0$ and $chs_1$ be the initial I/O env,
and $chs_0 = chs_1$
\[
  \begin{array}{rcl}
  chs_0 &=& \langle [], [\,], [true], [\,] \rangle \\[2mm]
  chs_1 &=& \langle [], [\,], [false], [\,] \rangle \\[2mm]
  \end{array}
  \]

When b = true, we enter the IF condition, and the security context is high. When we call $fprintf(stdout_lo,..)$, we will get SecErr according to our new Fprintf SecErr operational semantics.
By Sequence SecErr operational semantics, the entire program evalutes to a SecErrFrame error.

On the second run, if b = false, the we skip the IF condition, there is no wrong outout, so we just terminate the program normally, which evalute to a frame $(b \mapsto false^H)$ and chs, where the frame is not an error.

According to our extended definition of equivalence, frames only equal if they yield the same error. In our program with same starting states, 
one yields SecErrFrame, while the other yields a normal frame. Therefore by the new definition, because the result is not the same error(SecErrFrame), the program is not safe
